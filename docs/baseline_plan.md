# RebootRoute Baseline Plan

## 1. Objective
PDF 요구사항의 baseline model 항목을 충족하기 위해 RebootRoute는 dummy baseline과 실제 후보 모델을 같은 feature table과 같은 train/test split에서 비교합니다.

## 2. Data Split
- Source: `data/features/training_features.csv`
- Builder: `src/rebootroute/features/build_features.py`
- Split: `train_test_split(test_size=0.25, random_state=config.random_seed)`
- Stage label: `synthetic_stage_label`
- Mission label: `synthetic_mission_success`

현재 label은 synthetic MVP label입니다. 운영 환경에서는 실제 미션 완료, too-hard feedback, 프로그램 참여, 지원 결과, 운영자 review로 교체해야 합니다.

## 3. Stage Classifier Baseline
Task:

- 사용자 상태 feature를 기반으로 내부 stage를 분류합니다.

Models:

- `dummy_most_frequent`
- `logistic_regression`
- `random_forest`
- `gradient_boosting`

Metric:

- accuracy
- macro F1
- per-class precision/recall/F1
- confusion matrix

Selection:

- macro F1이 가장 높은 후보를 선택합니다.

## 4. Mission Success Predictor Baseline
Task:

- mission completion 가능성을 binary로 예측합니다.

Models:

- `dummy_stratified`
- `logistic_regression`
- `random_forest`
- `gradient_boosting`

Metric:

- accuracy
- macro F1
- ROC-AUC
- reliability summary
- confusion matrix

Selection:

- ROC-AUC가 있으면 ROC-AUC, 없으면 macro F1 기준으로 선택합니다.

## 6. Demo Interpretation
발표에서 모델 지표는 "운영 성능"이 아니라 "파이프라인이 baseline보다 의미 있는 신호를 학습하고 추적할 수 있는지"를 보여주는 증거입니다. 사용자 화면의 추천은 stage rule, resource 속성, 부담도 필터, 지도 거리 조건을 우선합니다.
